Overview

This library exposes the COVESA Vehicle Signal Specification as native Qt properties. Every VSS signal (speed, battery charge, HVAC temperature, etc.) becomes a Q_PROPERTY on a QObject-derived class, directly usable from C++ and QML with automatic change notifications.

12QML Modules
~50Interfaces
~300Properties
30+Enum Types
0Lines of Hand-Written C++

The entire library is generated from ~1,200 lines of QFace IDL and YAML annotations. No hand-written C++ exists for the signal classes. A simulator backend provides fake data for development without vehicle hardware. Production backends (CAN, SOME/IP, etc.) are swapped in at runtime without changing application code.

    TEMPLATE frontend
    IDL_IMPORT_PATH idl
)

# Backend simulator plugin
qt_ifcodegen_add_plugin(covesavss_vehicle_backend_simulator
    IDL_FILES idl/vehicle.qface
    TEMPLATE backend_simulator
    IDL_IMPORT_PATH idl

    Text { text: "Battery: " + battery.stateOfCharge.toFixed(0) + "%" }
}

Backend Plugin Discovery

Simulator plugins must be in an interfaceframework/ directory adjacent to the executable. During development:

# Symlink all simulator plugins next to the app binary
mkdir -p build/examples/dashboard/interfaceframework
ln -sf $(find build/src -name "libcovesavss_*_backend_simulator.dylib") \
       build/examples/dashboard/interfaceframework/

For production, replace the simulator .dylib files with your real backend plugin. No application code changes required.

Known Issues

Qt IF 6.10.2: Cross-Module Enum Stack Overflow

Using a cross-module enum type (e.g., Common.RoadSurfaceCondition) as a property type in a domain interface causes a stack overflow at runtime. The QIfSimulationEngine recursion guard fails to break the C++ ↔ QML call cycle for enum types from other modules.

Workaround: All 36 cross-module enum properties use int instead. Enum definitions are preserved in common.qface for documentation. See bugs/qtif-cross-module-enum-stackoverflow/ for the full analysis and investigation plan.

ifcodegen: Integer Defaults for Enum Properties

Specifying integer defaults for enum-typed properties in YAML files crashes the backend_simulator template at configure time with AttributeError: 'int' object has no attribute 'split'. Enum defaults must be omitted entirely.

Naming Constraints

Interface name ≠ module last segment Generated header self-include loop PowertrainStatus instead of Powertrain
