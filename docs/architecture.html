COVESA VSS Qt Library

Wraps the COVESA Vehicle Signal Specification (VSS) v6.0 as native Qt/QML types using Qt Interface Framework's ifcodegen code generator. No hand-written C++ signal classes — everything is generated from IDL.

Common Module — Shared Enums

All 36 enum types live in a single Common 1.0 QFace module. Domain modules import it via import Common 1.0. Values use type-specific prefixes to avoid collisions in the flat enum namespace.

How It Works at Runtime

1 QML Instantiates Types

  1. App imports a module: import COVESA.VSS.Vehicle
  2. Declares an interface: VehicleDynamics { id: dynamics }
  3. Binds to properties: dynamics.speed.toFixed(1) + " km/h"

2 Qt IF Loads Backend

  1. Frontend QObject asks Qt IF for a backend
  2. Qt IF scans interfaceframework/ for plugins
  3. Simulator plugin is loaded and connected

3 Simulator Provides Data

  1. Backend initializes with YAML defaults
  2. Properties emit change signals
  3. QML bindings update the UI automatically
